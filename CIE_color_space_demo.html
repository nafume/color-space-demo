RGB Additive

RGB는 검정에서 시작해 빛을 더하는 모델입니다. R/G/B 빛의 상대 세기가 커질수록 결과는 white에 가까워집니다.

CMY Subtractive

CMY는 흰 종이/조명에서 빛을 흡수하는 모델입니다. C는 red, M은 green, Y는 blue 성분을 줄이는 이상화된 필터로 표현합니다.

Transfer Function

RGB 코드값과 선형광/표시장치 밝기 사이의 변환 규칙입니다. 같은 RGB 숫자라도 이 곡선이 다르면 실제 빛의 비율이 달라집니다.

RGB Profile TRC

sRGB, Display P3, Adobe RGB, ProPhoto RGB 같은 RGB 프로파일은 고정 최대 nits보다 RGB code → linear RGB → XYZ 변환을 위한 TRC와 RGB↔XYZ matrix를 정의합니다.

ProPhoto RGB는 사진/인쇄 중심 ICC 워크플로와 PCS의 D50 기준에 맞추기 위해 D50 white point를 사용합니다.

OETF vs EOTF

OETF는 scene/light 값을 신호 코드로 압축하고, EOTF는 코드값을 디스플레이 빛으로 복원합니다. Rec.709/601/2020의 표시 실선은 BT.1886 감마 2.4 기준이며, PQ는 절대 nits 기반입니다.

왜 비선형 저장이 필요한가?
Linear 저장은 밝은 영역까지 같은 간격으로 code value를 배분하지만, gamma/log 저장은 사람이 더 민감한 shadow와 midtone에 더 촘촘한 단계감을 배치합니다.

Scene Exposure

0 stop은 18% gray입니다. +1 stop은 빛 2배, -1 stop은 빛 1/2배를 뜻하며 장면 안의 상대적인 노출 차이를 나타냅니다.

18% Gray

18% gray는 입사한 빛의 약 18%를 반사하는 중간 회색 표면입니다. 여기서는 scene-linear 0.18을 0 stop 기준으로 사용합니다.

Code Value

각 로그 곡선은 같은 노출값을 서로 다른 code value 위치에 배치합니다. black, middle gray, highlight headroom의 위치를 비교하세요.
