방송 색 표준 한눈에 보기

BT.601, BT.709, BT.1886, BT.2020, ST 2084(PQ), BT.2100, ARIB STD-B67(HLG)가 각각 무엇을 정의하는지, 그리고 영상 파일의 primaries / transfer / matrix 태그와 어떻게 연결되는지 비교한다.

읽는 법

표준 이름 하나가 색공간 전체를 항상 뜻하지는 않는다. 실제 파일은 보통 color_primaries, color_transfer, colorspace(matrix), color_range를 함께 확인해야 한다.

SD BT.601 matrix HD SDR BT.709 primaries + BT.709 matrix SDR 표시 BT.1886 reference EOTF UHD SDR BT.2020 primaries + BT.2020 matrix HDR10 BT.2020 + ST 2084(PQ) + BT.2020 NCL HLG BT.2020 + ARIB STD-B67 / BT.2100 HLG

색역과 밝기는 다른 축

BT.2020은 넓은 원색과 UHDTV 신호 체계를 정의한다. HDR 밝기 곡선인 PQ/HLG는 BT.2100 또는 ST 2084/ARIB STD-B67 쪽에서 봐야 한다.

BT.2084 표기는 주의

실무 UI에서 “BT.2084”라고 쓰는 경우는 대개 SMPTE ST 2084, 즉 PQ EOTF를 뜻한다. ITU-R HDR 시스템 표준은 BT.2100이다.

해상도만 믿지 않기

SD라서 무조건 BT.601, HD라서 무조건 BT.709라고 단정하면 위험하다. 파일 메타데이터와 제작 조건을 함께 확인해야 한다.

2. TV 제작 관점에서 보는 기준

SD / HD / UHD / HDR 제작 기준을 먼저 잡기

TV 제작과 프로그램 교환을 표준화하는 기준으로는 주로 ITU-R BT 계열을 보면 된다. SD는 BT.601, HD SDR은 BT.709, UHD/WCG는 BT.2020, HDR-TV 제작은 BT.2100이 중심이고, BT.1886, ST 2084, ARIB STD-B67은 각각 표시 EOTF나 HDR transfer처럼 더 좁은 축을 맡는다.

제작 상황 대표 표준 보면 되는 의미 함께 보는 표준 / 주의점
SD TV 제작 ITU-R BT.601 SD 디지털 컴포넌트 비디오의 색, Y'CbCr matrix, 샘플링, 코드 범위. 525-line과 625-line 계열의 역사적 색도 차이를 함께 확인한다.
HD SDR TV 제작 ITU-R BT.709 HDTV 제작/교환의 기본 색공간, OETF, Y'CbCr matrix, 1080계 시스템 파라미터. SDR 기준 모니터 표시 특성은 보통 BT.1886 또는 gamma 2.4 맥락과 함께 본다.
UHD SDR / WCG 제작 ITU-R BT.2020 UHDTV 해상도/프레임, Rec.2020 넓은 원색, RGB/YCbCr 신호 체계. BT.2020 primaries만으로 HDR이라고 판단하지 않는다. HDR 여부는 transfer를 같이 본다.
HDR-TV 제작 ITU-R BT.2100 HDR-TV 제작/교환용 이미지 파라미터. PQ와 HLG, BT.2020 원색, Y'CbCr/ICtCp. PQ 곡선은 ST 2084, HLG 핵심 transfer는 ARIB STD-B67과 연결된다.
SDR 기준 표시 ITU-R BT.1886 HDTV studio reference display가 따라야 할 SDR reference EOTF. 제작 시스템 전체가 아니라 기준 모니터의 표시 밝기 재현을 정의한다.
HDR PQ 밝기 곡선 SMPTE ST 2084 PQ 신호값을 절대 휘도 cd/m²로 매핑하는 HDR EOTF. 색역, matrix, 해상도/프레임 전체는 BT.2100 또는 BT.2020 쪽에서 본다.
HDR HLG 방식 ARIB STD-B67 Hybrid Log-Gamma HDR-TV transfer와 관련 시스템 파라미터. 전체 HDR-TV 제작/교환 표준으로는 BT.2100 HLG 맥락에서 함께 본다.

3. BT.601 이전의 TV 표준

아날로그 방송 시스템에서 디지털 스튜디오 표준으로

BT.601 이전에도 더 오래된 TV 표준들이 있었다. 다만 그 표준들은 지금의 Y'CbCr 파일 태그나 디지털 스튜디오 샘플링 표준이라기보다, 라인 수, 필드 주파수, 동기 신호, 컬러 부반송파, composite video 구조를 정한 아날로그 방송 시스템 표준에 가깝다.

시기 표준 / 시스템 성격 주로 정의한 것 BT.601과의 관계
1931 CIE 1931 색도 좌표계 색을 수학적으로 표시하는 기반 좌표계 RGB primaries와 white point를 나타낼 때 쓰는 (x, y) 색도 좌표. TV 제작 표준은 아니지만, BT.601/709/2020의 원색 좌표를 표현하는 바탕이다.
1941 NTSC 흑백 / 525-line System M 미국 525-line 아날로그 흑백 TV 전송 표준 525 lines, 30 frames/s, 60 fields/s 계열, 동기 신호, 방송 전송 구조. 이후 NTSC 컬러와 BT.601의 525-line SD 계열로 이어지는 역사적 기반이다.
1953 NTSC 컬러 흑백 TV와 호환되는 아날로그 컬러 방송 시스템 컬러 부반송파, YIQ 계열 색차 신호, 59.94 fields/s 계열 컬러 전송. BT.601의 525-line 계열 원색/매트릭스 관행에 영향을 준 아날로그 컬러 시스템이다.
1960s PAL 625/50 계열 중심의 아날로그 컬러 인코딩 시스템 위상 반전 방식의 컬러 부반송파, 색차 신호, PAL B/G/I/N/M 등 지역별 조합. BT.601의 625-line 계열 SD 제작/교환 맥락과 연결된다.
1960s SECAM 프랑스/동유럽권 등에서 쓰인 아날로그 컬러 인코딩 시스템 색차 성분을 순차적으로 보내는 FM 부반송파 방식, 625-line 계열 운용. PAL과 함께 BT.601의 625-line SD 계열 색도 관행을 이해하는 배경이 된다.
1970 계열 CCIR / ITU-R BT.470 conventional analogue television systems 권고 525/625-line 흑백 시스템과 NTSC, PAL, SECAM 컬러 TV 시스템의 주요 파라미터. BT.601이 디지털화하기 전의 아날로그 TV 시스템들을 정리해 보는 기준점이다.
1982 CCIR 601 / ITU-R BT.601 SDTV 디지털 스튜디오 컴포넌트 인코딩 표준 525/625-line SDTV를 디지털 Y'CbCr, 4:2:2/4:4:4, 샘플링, 코드 범위로 표현하는 방식. 아날로그 NTSC/PAL/SECAM 세계를 디지털 제작/교환 신호로 통일한 다리 역할을 한다.

따라서 “BT.601 이전 표준”을 볼 때는 NTSC/PAL/SECAM/BT.470을 먼저 떠올리면 된다. BT.601은 이 아날로그 SD 방송 세계를 그대로 composite로 보존한 표준이 아니라, 방송 제작과 국제 프로그램 교환을 위해 디지털 컴포넌트 신호로 다시 정리한 표준이다.

6. 흔한 혼동 정리

혼동 정리 실무 판단
BT.2020 = HDR? 아니다. BT.2020은 UHD/WCG 원색과 신호 체계를 정의한다. HDR transfer는 BT.2100의 PQ/HLG를 봐야 한다. primaries=bt2020만 보지 말고 transfer를 같이 확인.
BT.2084? 대개 잘못 줄여 부른 표현이다. 표준명은 SMPTE ST 2084이고, PQ EOTF를 뜻한다. 파일 태그는 보통 smpte2084. ITU-R HDR 시스템은 BT.2100.
BT.709 = sRGB? 원색과 D65 white는 같지만, 사용 맥락과 transfer, range, Y'CbCr matrix는 다르다. 이미지/웹 RGB는 sRGB, 방송/영상 신호는 BT.709 메타데이터와 range까지 확인.
BT.1886 = 파일 transfer tag? 대개 아니다. BT.1886은 HDTV studio reference display가 따라야 할 EOTF다. 파일은 보통 transfer=bt709로 남고, 모니터링/컬러 매니지먼트에서 BT.1886 또는 gamma 2.4 계열을 선택한다.
해상도만으로 matrix 결정? 전통적으로 SD는 BT.601, HD는 BT.709, UHD는 BT.2020이 흔하지만 예외가 있다. 업스케일 SD, 편집본, 잘못 태그된 파일은 반드시 메타데이터와 눈검사를 함께 한다.
HLG = ARIB STD-B67만 보면 끝? ARIB STD-B67은 HLG의 핵심 파라미터를 정의했고, BT.2100은 HLG를 HDR-TV 시스템 안에 통합한다. NLE에서는 Rec.2100 HLG, 파일 태그에서는 arib-std-b67로 보일 수 있다.

7. HDR 메타데이터 표준

정적 / 동적 / 전달 방식 분리

BT.2100, ST 2084, ARIB STD-B67은 HDR 신호와 transfer를 정의하는 표준이고, ST 2086, CTA-861.3, ST 2094 계열은 HDR 메타데이터를 정의하거나 전달하는 표준이다. 실제 파일에서는 이 메타데이터가 코덱 SEI, HDMI InfoFrame, 컨테이너 박스, Dolby Vision RPU 같은 서로 다른 위치에 실릴 수 있다.

구분 표준 / 규격 정의하는 메타데이터 주로 보이는 위치 대표 포맷 / 실무 메모
정적 메타데이터 SMPTE ST 2086 Mastering Display Color Volume. 마스터링 디스플레이의 RGB primaries, white point, minimum/maximum luminance.
HEVC SEI MP4/MOV metadata MKV metadata
HDR10에서 가장 자주 보이는 정적 mastering display metadata.
정적 메타데이터 / HDMI CTA-861.3 / CEA-861.3 HDR Static Metadata Extension. EOTF capability, static metadata descriptor, HDR Static Metadata InfoFrame, EDID CTA data block.
HDMI InfoFrame EDID CTA block
TV가 어떤 HDR EOTF와 정적 메타데이터를 받을 수 있는지 신호화하는 데 중요하다.
콘텐츠 밝기 정보 MaxCLL / MaxFALL Maximum Content Light Level, Maximum Frame-Average Light Level. 전체 콘텐츠의 최대 픽셀 밝기와 최대 프레임 평균 밝기.
HDR10 static metadata Content light level SEI
톤매핑 힌트로 자주 쓰이지만, 값이 없거나 부정확한 파일도 많다.
코덱 내 전달 ITU-T H.265 / ISO/IEC 23008-2 HEVC SEI Mastering display colour volume SEI, Content light level information SEI 등 HDR 메타데이터를 비트스트림 안에 싣는 메시지 구조.
HEVC elementary stream SEI NAL unit
ST 2086/MaxCLL/MaxFALL 값을 실제 H.265 스트림에 실을 때의 대표 전달 경로.
동적 메타데이터 SMPTE ST 2094 suite Dynamic Metadata for Color Volume Transform. 장면/프레임별 color volume transform과 display mapping에 필요한 공통 구성요소와 application별 데이터.

ST 2094-1은 core components, ST 2094-10/20/30/40 등은 application별 정의.
HDR10+ SMPTE ST 2094-40 Dynamic Metadata for Color Volume Transform Application #4. HDR10+에서 쓰이는 장면/프레임 단위 동적 메타데이터.
HEVC SEI AV1 metadata HDR10+ JSON/tooling
MediaInfo에서 SMPTE ST 2094 App 4처럼 보이는 경우가 많다.
Dolby Vision 관련 Dolby Vision metadata / RPU, ST 2094-10, ETSI TS 103 572 Dolby Vision의 global/shot/frame 기반 metadata와 RPU. 공개 표준으로는 ST 2094-10 Application #1 및 DVB 전달 규격인 ETSI TS 103 572와 연결된다.

HLG ARIB STD-B67 / BT.2100 HLG HLG 자체는 HDR10식 mastering display metadata나 동적 metadata에 의존하지 않는 방송 친화 HDR 방식.
transfer=arib-std-b67 Rec.2100 HLG signaling
HLG 파일에 ST 2086/HDR10 metadata가 붙어 있으면 제작/인코딩 의도를 따로 확인해야 한다.

실무적으로는 HDR10이면 bt2020 + smpte2084 + bt2020nc 색 신호 태그에 ST 2086과 MaxCLL/MaxFALL이 붙는 경우가 많고, HDR10+는 여기에 ST 2094-40 동적 메타데이터가 추가된다. Dolby Vision은 RPU metadata를 따로 보며, HLG는 보통 정적 metadata보다 transfer signaling이 더 중요하다.
