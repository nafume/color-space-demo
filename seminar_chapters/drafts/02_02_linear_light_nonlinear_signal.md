# [Draft] 2회차 Chapter 2. 선형광(Linear Light)과 비선형 신호(Non-Linear Signal)

## 학습 목표

이 장의 목표는 물리적 빛은 선형적(linear)으로 더해지지만, 이미지와 영상 신호는 대개 비선형(non-linear)으로 저장된다는 사실을 이해하는 것이다. 선형광(linear light)과 비선형 RGB(non-linear RGB)를 구분하지 못하면 합성, 리사이즈, 블러, 색공간 변환, 노출 조정에서 예상과 다른 결과가 나온다.

이 장을 마치면 청중은 선형 RGB(linear RGB)와 비선형 RGB(non-linear RGB)의 차이, 사람이 어두운 영역 변화에 더 민감한 이유, 그리고 왜 대부분의 일반 이미지 파일이 선형광 그대로 저장되지 않는지 설명할 수 있어야 한다.

## 핵심 질문

- 물리적 빛의 선형성(linearity)이란 무엇인가?
- 인간 시각(human vision)은 왜 빛의 세기에 선형적으로 반응하지 않는가?
- 이미지를 선형광(linear light)으로 저장하면 항상 좋은가?
- 선형 RGB(linear RGB)와 비선형 RGB(non-linear RGB)는 어떻게 다른가?
- 어떤 처리는 선형광에서 해야 하고, 어떤 값은 표시용 비선형 신호로 보아야 하는가?

## 상세 설명

### 1. 물리적 빛은 선형적으로 더해진다

물리적 빛의 에너지는 기본적으로 선형적(linear)으로 더해진다. 같은 조명이 하나 있을 때보다 같은 조명이 두 개 있을 때, 이상적인 조건에서는 들어오는 빛의 양이 두 배가 된다. 디스플레이(display)의 발광도 물리적으로는 특정 채널의 빛을 더해 목표 자극을 만드는 과정이다.

그래서 실제 빛의 양을 계산해야 하는 작업은 선형광(linear light)에서 처리하는 것이 자연스럽다. 예를 들어 두 색을 반반 섞거나, 이미지를 블러 처리하거나, 반투명 레이어를 합성할 때는 코드값(code value)을 그냥 평균내는 것보다 선형광으로 변환한 뒤 계산하는 편이 물리적으로 더 맞다.

단순 예시를 보자. 비선형 sRGB 값 `0.5`는 물리적 빛의 50%가 아니다. sRGB 전송 곡선을 역으로 풀면 선형광 값은 대략 0.214 근처다. 따라서 sRGB 코드값에서 `0.5 + 0.5`처럼 생각하면 실제 빛의 양과 맞지 않는다.

### 2. 인간 시각은 비선형적으로 민감하다

사람은 빛의 물리적 세기에 선형적으로 반응하지 않는다. 어두운 영역에서는 작은 차이도 비교적 잘 느끼고, 아주 밝은 영역에서는 같은 물리적 차이를 덜 민감하게 느끼는 경향이 있다. 이것을 넓게 말하면 인간 시각의 비선형 민감도(non-linear sensitivity)라고 할 수 있다.

이 특성 때문에 제한된 비트 수(bit depth)로 이미지를 저장할 때 선형광 값을 그대로 균등하게 나누면 비효율적이다. 예를 들어 8비트(8-bit)로 0부터 255까지 코드를 배분한다고 할 때, 선형광으로 균등 배분하면 사람이 민감한 어두운 영역에 충분한 코드가 배정되지 않아 계조(gradation)가 거칠어 보일 수 있다.

비선형 전송 함수(transfer function)는 이런 문제를 완화한다. 코드값을 사람의 지각 민감도에 더 맞게 배분해, 같은 비트 수에서도 눈에 띄는 밴딩(banding)을 줄이고 저장 효율을 높인다.

### 3. 왜 이미지는 보통 선형광 그대로 저장되지 않는가

일반적인 SDR(Standard Dynamic Range) 이미지와 영상은 대개 선형광 그대로 저장되지 않는다. 이유는 크게 세 가지다.

첫째, 저장 효율이다. 8비트나 10비트 같은 제한된 코드값을 사람이 민감하게 느끼는 구간에 더 많이 배분하는 편이 시각적으로 유리하다.

둘째, 역사적 호환성이다. CRT(Cathode Ray Tube) 디스플레이는 입력 전압과 출력 빛 사이에 자연스러운 비선형 관계가 있었고, 방송과 영상 표준은 이 특성을 전제로 발전했다. 오늘날의 디스플레이는 CRT가 아니지만, 기존 신호 체계와 호환되는 전송 함수가 여전히 중요하다.

셋째, 표시 파이프라인(display pipeline)의 관습이다. 카메라, 편집툴, 파일 포맷, 코덱(codec), 디스플레이는 코드값과 빛 사이의 관계를 특정 전송 특성(transfer characteristics)으로 약속한다. 이 약속을 알아야 숫자를 올바르게 해석할 수 있다.

### 4. 선형 RGB와 비선형 RGB를 구분해야 한다

선형 RGB(linear RGB)는 각 채널 값이 물리적 빛의 양에 비례하도록 표현된 RGB다. 예를 들어 선형 RGB에서 `0.5`는 기준 흰색에 대해 절반의 선형광을 뜻한다.

비선형 RGB(non-linear RGB)는 전송 함수가 적용된 RGB다. sRGB 이미지 파일의 픽셀값이나 Rec.709 영상의 코드값은 보통 표시나 저장을 위한 비선형 신호다. 비선형 RGB에서 숫자 `0.5`는 물리적 빛 50%를 뜻하지 않는다.

실무에서 흔한 오류는 비선형 RGB 값을 선형 값처럼 계산하는 것이다. 예를 들어 sRGB 코드값에서 바로 평균을 내면 중간색이 물리적으로 어둡게 나올 수 있다. 올바른 흐름은 보통 다음과 같다.

```text
비선형 RGB
-> inverse transfer
-> 선형 RGB
-> 물리적/색공간 계산
-> transfer 적용
-> 비선형 RGB
```

## 용어 노트

### 선형광(Linear Light)

선형광(linear light)은 값이 실제 빛의 양에 비례하는 표현이다. 합성, 필터링, 조명 계산, 색공간 변환의 중간 단계에서 중요하다.

### 비선형 신호(Non-Linear Signal)

비선형 신호(non-linear signal)는 사람의 지각, 저장 효율, 표시 장치 특성 등을 고려해 빛의 양과 코드값이 직선 관계가 아니도록 만든 신호다.

### 선형 RGB(Linear RGB)

선형 RGB(linear RGB)는 RGB 채널 값이 선형광에 비례하는 상태다. 같은 원색과 화이트 포인트를 쓰더라도 전송 함수를 풀었는지에 따라 선형 RGB와 비선형 RGB가 나뉜다.

### 비선형 RGB(Non-Linear RGB)

비선형 RGB(non-linear RGB)는 sRGB 곡선, Rec.709 OETF 같은 전송 함수가 적용된 RGB 신호다. 저장과 표시에는 유리하지만 물리적 계산에는 그대로 쓰면 안 되는 경우가 많다.

### 코드값(Code Value)

코드값(code value)은 파일이나 영상 신호에 저장된 디지털 숫자다. 코드값은 전송 함수와 색 범위(color range)를 함께 알아야 실제 빛 또는 색 의미로 해석된다.

## 실무 예시와 데모 아이디어

### 예시 1. sRGB에서 바로 평균낸 회색과 선형광 평균 비교

검정과 흰색을 50%로 섞는 예를 보여준다. sRGB 코드값에서 바로 평균내면 `128` 근처가 나오지만, 선형광 관점에서 중간 빛을 다시 sRGB로 인코딩하면 더 밝은 코드값이 된다. 이 차이는 "코드값 중간"과 "빛의 중간"이 다르다는 점을 보여준다.

### 예시 2. 비선형 공간 블러와 선형광 블러 비교

밝은 글자나 하이라이트가 있는 이미지를 두 방식으로 블러 처리한다. 비선형 RGB에서 블러하면 가장자리 에너지가 어색하게 어두워질 수 있고, 선형광 블러는 빛의 확산에 더 가까운 결과를 낸다.

### 예시 3. PNG/JPEG와 OpenEXR 비교

일반 PNG/JPEG는 보통 sRGB 같은 비선형 표시 신호에 가깝고, OpenEXR은 제작 워크플로에서 선형 장면 참조(scene-referred) 데이터로 많이 쓰인다는 점을 비교한다.

## 추천 진행 흐름

### 1. "숫자 0.5는 빛 50%인가?"로 시작하기

청중에게 sRGB 값 `0.5`가 빛의 절반인지 질문한다. 이 질문으로 코드값과 빛의 양이 다를 수 있다는 문제를 열어 준다.

### 2. 물리적 빛의 선형성 설명하기

빛의 에너지는 선형적으로 더해진다는 점을 설명한다. 합성, 블러, 리사이즈가 왜 선형광에서 자연스러운지 연결한다.

### 3. 인간 시각의 비선형 민감도로 넘어가기

사람은 어두운 영역 변화에 더 민감하므로, 저장 신호는 비선형 배분이 효율적이라고 설명한다.

### 4. 선형 RGB와 비선형 RGB를 도식으로 분리하기

같은 RGB라도 전송 함수 적용 전후가 다르다는 점을 보여준다. 이후 장의 OETF와 EOTF로 자연스럽게 연결한다.

## 짧은 마무리 요약

빛은 물리적으로 선형적으로 더해지지만, 인간 시각은 밝기 변화에 비선형적으로 반응한다. 그래서 일반 이미지와 영상은 저장 효율과 지각 품질을 위해 선형광(linear light) 그대로 저장되지 않는 경우가 많다.

핵심은 선형 RGB(linear RGB)와 비선형 RGB(non-linear RGB)를 구분하는 것이다. 계산은 선형광에서 해야 하는 경우가 많고, 저장과 표시에는 전송 함수가 적용된 비선형 신호가 쓰인다.
