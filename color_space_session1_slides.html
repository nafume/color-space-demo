Opening Question

같은 RGB 값은 항상 같은 색일까?

1회차의 목표는 RGB 숫자를 그대로 믿지 않고, 그 숫자가 어떤 표준과 좌표계 안에서 해석되는지 설명할 수 있게 되는 것이다.

  • 색값(color value)과 색공간(color space)을 분리한다
  • CIE xy 색도도에서 색역(gamut)을 읽는다
  • XYZ와 Lab이 왜 변환의 공통 언어인지 이해한다
  • ICC Profile과 표준 metadata의 역할을 구분한다
  • 마지막에는 실제 색공간 변환 실습으로 연결한다
Web Standard Demo

같은 RGB 값, 다른 Color Space

CSS의 color() 함수는 같은 3채널 값을 특정 색공간 안에서 해석할 수 있다. 아래에서 같은 숫자를 여러 웹 표준 색공간으로 비교해 본다.

R 255
G 0
B 0

rgb()와 #hex는 기본적으로 sRGB다. 선택지는 CSS 표준 색공간이며, 미구현 항목은 자동으로 비활성화된다.

sRGB
color(srgb 1 0 0)
Display P3
color(display-p3 1 0 0)

Chapter 2

색은 빛의 파장만으로 끝나지 않는다

색공간은 물리적 스펙트럼과 인간 관찰자의 반응을 함께 다룬다. 그래서 CIE 표준 관찰자와 색상 매칭 실험이 출발점이 된다.

Trichromatic

L, M, S 원추세포의 민감도 차이가 색 지각의 생리적 기반이 된다.

Color Matching

관찰자가 기준 색과 세 기본 빛의 조합이 같아 보인다고 판단한 결과를 표준화한다.

Chapter 3

XYZ는 계산의 언어, xy는 색도의 지도

CIE XYZ는 장치 독립 좌표이고, CIE xy는 XYZ를 정규화해 밝기 크기를 제거한 색도 좌표다.

x = X / (X + Y + Z) y = Y / (X + Y + Z) z = Z / (X + Y + Z) Y = 상대 휘도(relative luminance) y = 색도 좌표(chromaticity coordinate)
Chapter 4

xy에서 Lab으로: 색차를 숫자로 말하기

CIE Lab은 XYZ와 기준 흰색을 바탕으로 색을 명도와 반대색 축에 배치해, 사람이 느끼는 색 차이를 더 다루기 쉽게 만든다.

  • L*: 명도(lightness), 기준 흰색에 대한 상대 지각 척도
  • a*: 초록에서 빨강으로 향하는 반대색 축
  • b*: 파랑에서 노랑으로 향하는 반대색 축
  • Delta E는 Lab 좌표 차이를 실무 품질 기준으로 바꿔 준다
Chapter 5

색도도에서 표준을 읽는 법

sRGB, Display P3, Rec.2020은 이름이 아니라 원색과 화이트 포인트를 가진 표준이다. CIE xy는 그 관계를 한 장의 지도처럼 보여준다.

Chapter 7

입력 RGB의 의미를 해석하고, 장치 독립 좌표를 거쳐, 대상 색공간의 규칙으로 다시 표현한다.

Source RGB · encoded values

RGB to XYZ · device independent
Chromatic adaptation · if white point changes
Target RGB · apply target transfer
Chapter 8

RGB와 CMYK는 같은 문제를 다르게 푼다

디스플레이는 빛을 더하고, 인쇄는 잉크가 빛을 흡수한다. CMYK 변환은 RGB의 단순 반대가 아니라 인쇄 조건의 결과다.

Additive

RGB는 발광 장치에서 빛을 더해 색을 만든다.

Subtractive

CMYK는 종이와 잉크, 총잉크량, black generation의 제약을 받는다.

Chapter 9-10

ICC Profile은 장치 색을 공통 언어로 연결한다

Source profile과 target profile은 PCS(Profile Connection Space)를 통해 연결된다. PCS는 CIE XYZ 또는 CIE Lab 기반이며 D50 기준이 중요하다.

Source device values -> Source ICC Profile -> PCS: CIE XYZ or CIE Lab -> Target ICC Profile -> Target device values
Chapter 11

표준 이름보다 정의 항목을 읽는다

PPrimaries: red, green, blue 좌표
WWhite point: D65, D50 같은 흰색 기준
TTransfer: 코드값과 빛의 관계
M/RMatrix, Range: 영상 신호 해석
Chapter 12

실습은 변환 결과보다 해석 과정을 본다

같은 RGB 값, assign vs convert, gamut clipping, RGB to CMYK를 직접 비교하며 색공간 변환의 중간 단계를 확인한다.

  • 프로파일을 확인한다
  • 색공간을 할당(assign)해 본다
  • 색공간을 변환(convert)해 본다
  • 변환 전후 RGB 숫자와 외관을 비교한다
  • gamut warning과 soft proofing으로 마무리한다
